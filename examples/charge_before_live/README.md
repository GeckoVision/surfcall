# Charge-before-live — Surfpool-for-APIs

> **Surfpool forks the chain to test your program; Gecko comprehends + simulates
> the API your program calls.**

[Surfpool](https://github.com/txtx/surfpool) gives Solana builders a forked mainnet
so they can run a program against real chain state *before* they deploy. This example
is the API-side complement: point Gecko at a payment API and it verifies the **charge
call is first-call-correct** — right unit, required idempotency key, bearer auth —
**offline, $0, before a cent moves.**

Here the API is a small payment-API stub (`spec/payapi_openapi.json`). It encodes the
failure modes that make real payment APIs painful:

| Trap | How the spec encodes it | How Gecko verifies it |
|---|---|---|
| **Unit trap** — amount is **integer centavos**, not dollars (R$5.00 = `500`) | `amount: { type: integer }` + description; required | The unit is *surfaced* to the agent in the tool schema/description (comprehension), and the synthesized first call carries it as an integer |
| **Idempotency key required** — a retry must not double-charge | `Idempotency-Key` header, `required: true` | Required-field guard: dropping it raises `CallError` **locally**, before any wire call |
| **Bearer auth** | `bearerAuth` security scheme | Auth is hidden from the agent and injected at call time; the bearer-gated tool only surfaces when a session can satisfy it |

## What this proves

Run in **recorded** mode, the engine synthesizes a valid response from the schema —
no network, no keys, no spend — and checks the call the agent *would* make:

1. **Discovery** — `client.search("charge a customer")` returns the charge endpoint
   (ranked first), with no API docs in front of the model.
2. **Well-formed first call** — synthesized valid args → `2xx` + data (`id`, `status`,
   integer `amount`).
3. **Required-field guard** — drop the idempotency key (or `amount`) and the engine
   raises `gecko.caller.CallError` *before* a request is ever sent.
4. **`testgen.check`** — the shipped generator runs both checks for every usable tool:
   **4/4 pass** (2 tools × {well-formed, required-guard}).

### Honest scope

The recorded check verifies **presence + shape + auth wiring**: the required fields are
there, the body is the right shape, a `2xx` comes back, and auth is injected — caught
offline instead of fired live. It does **not** semantically reject a wrong-unit value
(e.g. a float `5.00` instead of `500`); that's why the unit lives in the tool
**description and schema type** so the agent fills it correctly. This is comprehension +
structural simulation, not a runtime money-validator.

No real charge is made. No real secret is used — the example injects a throwaway stub
bearer token via the documented session seam (`auth_headers()`).

## Run it ($0, offline)

```bash
uv run pytest examples/charge_before_live/ -q
```

10 tests, fully offline:
- `test_charge_first_call_correct.py` — 6 hand-written checks (discovery, auth hidden +
  unit surfaced, recorded well-formed call, two required-field guards, `testgen` 4/4).
- `test_payapi_firstcall.py` — auto-generated by `gecko.testgen.render_module`
  (parametrized well-formed + required-guard per tool). Regenerate when the API drifts;
  run it from the repo root.

## Local vs cloud (honest)

- **Local = free — and that's the whole of this demo.** Recorded mode + `testgen` are
  **shipped today**. Comprehend a spec, simulate the call, and gate CI on
  first-call-correctness for $0, offline. That is everything you see here.
- **Cloud = paid — not in this repo.** Continuous **live** verification against the real
  API, drift detection / auto-regeneration when the spec or behavior changes, and
  fleet-scale analytics are the **V2 / hosted** story. They are *not* implemented here;
  treat any mention of them as roadmap, not a claim.

| Capability | Status |
|---|---|
| Recorded-mode first-call simulation ($0, offline) | **Live (this demo)** |
| `testgen.check` / `render_module` | **Live (this demo)** |
| Continuous **live** verification at scale | V2 / cloud (not in this repo) |
| Drift detection + auto-regeneration | V2 / cloud (not in this repo) |
| Fleet analytics | V2 / cloud (not in this repo) |
